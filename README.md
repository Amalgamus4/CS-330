How do I approach designing software?

    What new design skills has your work on the project helped you to craft?
      I have learned how to consider lighting in design and how it can impact a scene. Implementing the lighting is much more involved than I anticipated, and I
      still have a lot to learn on the subject.
    What design process did you follow for your project work?
      I used an interative approach, beginning with a single object and slowly building the scene, piece by piece.
    How could tactics from your design approach be applied in future work?
      An iterative design approach is always a good choice when working on larger, complicated systems. This is especially true for software engineering. 
      By startting small and gradually working towards the end goal, it is much easier to spot and fix mistakes as they arise.

How do I approach developing programs?

    What new development strategies did you use while working on your 3D scene?
      One strategy I used was to reuse code where possible. By this, I mean I would create an object, then realize I needed a simlar object, just in a different
      place within the scene, so I would copy the code, and make an adjustment, say to the tranlastion of the object. This made it easier to add new objects to
      the scene without having to completely rewrite the code for each new object.
    How did iteration factor into your development?
      I used a highly iterative approach, constantly making small changes and viewing the results, then making adjustments as needed for each piece of the scene.
    How has your approach to developing code evolved throughout the milestones, which led you to the project’s completion?
      I am beginning to see more and more how modularity in code is cruicial for large, maintainable systems. I will be using this principle as much as I can
      in my future work.
